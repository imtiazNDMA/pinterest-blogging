# Playbook: Pin Creation (Pinterest Traffic System)

Used by **pinterest-strategist** / `/make-pins`. Turn every post into multiple fresh Pinterest tests.

## Per post
- Create **4–6 pins**, each a DIFFERENT angle (search / curiosity / product / budget / beginner / seasonal /
  list). Different angle = different design = a **fresh pin** (not a re-share).
- Each pin: 2:3 / 1000×1500, bold thumbnail-legible overlay, keyword-rich title + varied description,
  keyword-named board, suggested filename, destination = the blog post.
- Assign a `Pin Batch ID` (PB-###) and a **spaced schedule** (e.g., 1/day across the batch, evening slots).

## Workflow
1. `/make-pins <slug>` → pinterest-strategist fills `templates/pin-batch.md` → `Content/pins/<slug>-pins.md`.
2. User designs each pin in **Canva** (free) from the briefs, using original `[YOUR PHOTO]` images where possible.
3. User schedules via the **Pinterest native scheduler**, spaced out.
4. Append `Pin` rows to the tracker; update Pin Status as they publish.

## Cadence (account-wide)
- ~5–10 fresh pins/day once warmed up; ramp gradually on a new account (don't spike from zero).
- A few Idea/video pins per week for algorithm trust (not clickable — reach/authority, not direct traffic).
- Strong window ≈ 8–11 PM audience timezone (validate later with your own analytics).

## Optimization tie-in
Every 2–4 weeks (`/weekly-review`): which pin titles get saves? which designs get impressions but no clicks
(redesign/retire)? which boards gain traction? Feed winners back into more posts + more angles.
