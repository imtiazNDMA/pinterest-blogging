# Guardrail: Pinterest Policy & Best Practice (2026)

**Status: hard rules.** Pinterest is a visual search engine. Treat it as SEO, not social. Re-verify
specifics (the algorithm shifts); facts tracked in `.claude/memory.md`.

## Fresh pins (the golden rule)
- Pinterest rewards **fresh pins** = a new image/design, not a repin of an existing graphic.
- Each blog post gets **multiple distinct pin designs** over time, not the same pin re-shared.
- Spread publishing: schedule pins **spaced out** (native scheduler), never dump all at once.

## Format
- **Strict 2:3 vertical ratio — 1000×1500 px.** Square and horizontal underperform; do not use them.
- Bold, legible text overlay (readable at thumbnail size); high-contrast; brand consistency.
- Keyword everything: **pin title, description, board name, and image filename**.

## Anti-spam / trust
- **Pins must match their destination** — no misleading titles or bait. The linked post must deliver.
- No excessive/irrelevant hashtags; a few relevant keywords in description is enough.
- No identical descriptions copy-pasted across many pins; vary naturally.
- Don't link pins straight to raw affiliate URLs as a default — link to the blog post (better UX, fewer policy issues, and the post is monetized anyway). Disclose if a pin is itself an ad.
- Build **topical authority**: keep boards and pins consistent with the niche entity (e.g., container gardening) so the account is trusted.

## Cadence & mix
- ~5–10 fresh pins/day, spaced; ramp gradually on a new account (don't spike).
- A few **Idea/video pins per week** for algorithm trust (note: not clickable — use for reach/authority, not direct traffic).
- Strong posting window ≈ 8–11 PM in the **audience's** timezone (validate with your own analytics later).

## Boards
- Keyword-named boards, one per content pillar + sub-topic boards as the library grows.
- Board descriptions are keyword-rich and on-topic.

## pinterest-strategist checklist
- [ ] 2:3 / 1000×1500 specified in every design brief
- [ ] Multiple distinct angles per post (fresh designs)
- [ ] Keyword-rich title + description + suggested filename + board
- [ ] Destination = relevant blog post; pin promise matches post
- [ ] Spaced schedule, not a bulk dump
